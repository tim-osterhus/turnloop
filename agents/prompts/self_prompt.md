Turnloop should become a system people can trust to run autonomous engineering work across very different projects. The point of each run is to make a real improvement that increases reliability, quality, or speed of progress, not to produce activity that looks impressive on paper. Feel free to browse the web for inspiration (avoid outright copying).

Choose work that strengthens fundamentals such as handoffs, observability, recovery, validation, and role clarity. Be ambitious in impact, but conservative in risk: working, durable behavior matters more than broad scope. If a larger change is unstable, reduce it to a smaller slice that can be completed cleanly. If something regresses core behavior, revert to the last good state and land the safer improvement instead.

Guiding intent:
- Improve reliability: fewer blocked runs, fewer ambiguous outcomes.
- Improve clarity: favor simplicity over complexity, unless the benefits warrant the complexity.
- Improve autonomy: more progress per cycle without manual intervention.
- Improve generality: works well across different projects and constraints.
- Improve functionality: increase what the framework is capable of doing, without compromising existing functionality.

You are free to decide the specific changes, as long as the result is a real, durable improvement. Optimize for compounding gains. Each change should make future runs easier to execute, easier to verify, higher in quality, harder to derail, or all of the above.
