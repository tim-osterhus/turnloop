Build a lightweight but in-depth browser‑based mining game inspired by MotherLoad. The long‑term aim is a smooth, satisfying game that stays fast and stable even as more content is added. The experience should feel simple to pick up but meaningful to master: descend, gather, return, upgrade, repeat. Each loop should give the player a clear sense of progress and a reason to go deeper. Feel free to browse the web for inspiration (avoid outright copying).

Corebound should steadily become a standout mining game with tight controls, clear feedback, meaningful progression, and dependable performance. Every run should ship at least one improvement players can substantively feel, whether that is better moment-to-moment play, nicer visuals, clearer progression, stronger pacing, greater content depth, or better stability. Prefer improvements that reduce friction for new players, unless you deem the improvement sufficiently beneficial to warrant that increased friction.

Aim high on quality, not size. A smaller change that is complete, tuned, and robust is better than a wide change that is fragile or introduces bugs. Keep the game runnable throughout. If a change breaks core play, roll back to the last good state and deliver a safer version.

New ideas should reinforce the core loop of descend, mine, return, and upgrade. If there is no playable baseline, create one first and then improve it. The goal is a game that keeps getting better without ever becoming brittle. If you introduce new mechanics, make sure they integrate naturally with the existing loop. Variety matters, but only if it strengthens the core loop and keeps the game readable and coherent.
