# Millrace Theme

This note describes the shared visual theme for Millrace public pages. It is a local guidance artifact for agents working on `game.millrace.ai` and related public-facing subpages.

## Core Feel

- Dark, high-contrast, and deliberate.
- Polished, but still experimental and slightly industrial.
- More "autonomous lab building in public" than generic startup dashboard.

## Shared Shell Contract

- Public-facing game pages should feel like pages inside the Millrace arcade, not unrelated microsites.
- Keep a recognizable Millrace shell where possible: branded top bar, clear backlink to the arcade, rounded panel framing, pill-style metadata, and restrained glow accents.
- Use expressive display typography for major headings, readable sans-serif for body copy, and monospace sparingly for pills, versions, or telemetry.

## Color And Material

- Prefer deep charcoal or near-black backgrounds over light themes.
- Use layered panel surfaces, subtle gradients, glass-like depth, and soft borders instead of flat blocks.
- Keep the outer shell close to the Millrace brand language: dark neutrals, warm red accents, and restrained highlight glow.
- Game-specific accent colors are welcome inside the play surface, HUD, and game-specific UI.

## What Can Vary

- The game field, HUD, particles, and in-world feedback can have their own identity.
- A mining game does not need to look like a racing game; only the shell needs to remain recognizably Millrace.
- New games can introduce a secondary accent palette as long as the page still reads as part of the same family.

## Avoid

- Unrelated font stacks that make the page feel detached from Millrace.
- Flat white backgrounds or generic SaaS-style app framing.
- Removing the arcade backlink or hiding the page inside a shell with no Millrace cues.
- Replacing the whole shell just to make the game interior feel distinct.

## Practical Rule Of Thumb

Keep the outer page frame cohesive and branded. Let the actual game area carry the unique mood.
